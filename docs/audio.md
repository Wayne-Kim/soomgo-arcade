# 오디오 — 효과음 및 출처

게임의 **오디오 레이어** 및 포함된 모든 사운드의 **독창성/라이선스 출처**에 대한 단일 진실 공급원(SSOT)이며, [`BRAND.md`](BRAND.md)에 명시된 브랜드의 독창성 입장을 따릅니다.

## 범위

- **포함 사항:** 핵심적인 게임 상황에 재생되는 렌더링 측 효과음 및 지속 유지되는 볼륨/음소거 조작 기능.
- **제외 사항:** 배경 음악 트랙 및 아나운서/캐릭터 음성 대사 (리소스 제작 비용 및 다국어 번역 비용을 절감하기 위해 의도적으로 제외된 항목입니다).

## 아키텍처 (렌더링 측 전용)

오디오 레이어는 자동 로드되는 싱글톤 오브젝트인 `Audio` (`scripts/audio/Audio.gd`)로 구현되어 있습니다. 이는 **렌더링/UI 측에서 결정론적 시뮬레이션의 기존 시그널을 감지하여 구동**됩니다 (`scripts/ui/Game.gd::_connect_audio` 및 카운트다운/결과 처리기 참고). 오디오 구현 시 아래의 엄격한 규칙을 준수해야 합니다:

- 고정 틱 단위로 실행되는 `Simulation.step()` 내부에서는 어떠한 오디오 호출도 수행하지 않습니다. 시뮬레이션 코드는 오디오 상태를 읽지도 생성하지도 않으므로, **록스텝 결정론(lockstep determinism)에 전혀 영향을 주지 않으며** 네트워크 동기화 테스트도 문제없이 통과합니다.
- 네트워크 멀티플레이 시, 각 피어(Peer)는 렌더링 측 시그널 처리기에 개별적으로 도달하여 **자신의 로컬 기기에서 오디오를 재생**합니다 — **오디오 데이터는 네트워크를 통해 전혀 전송되지 않습니다**.
- 오디오는 항상 기존의 시각적 피드백(보드 깜빡임, 오버레이, 카운트다운 숫자)을 **보완하는 역할**을 하며, 어떠한 게임 상태도 **오디오만 단독 정보로 제공되어서는 안 됩니다**. 이는 단일 정보 소스에 의존하지 않는다는 프로젝트의 접근성 원칙을 따릅니다.

### 다성음(Polyphony), 헤드리스 및 설정 유지

- **다성음 제한:** `Audio.VOICES` (12개)로 크기가 고정된 `AudioStreamPlayer` 풀을 사용합니다. 대규모 연쇄 폭발 발생 시 동시에 수많은 시그널이 발생하는데, 오디오 재생 요청이 풀의 한도를 넘어서면 음이 깨지거나 소리가 너무 뭉개지는 것을 방지하기 위해 새로운 사운드를 겹쳐 쌓는 대신 가장 오래된 음성을 재사용합니다.
- **전용 버스:** 효과음은 런타임에 생성되는 `SFX` 버스에서 재생되며 `Master` 버스로 라우팅됩니다. 이를 통해 볼륨/음소거 제어가 `Master` 버스를 건드리지 않고 개별 작동하므로, 향후 음악용 버스를 추가할 공간을 남겨둘 수 있습니다.
- **헤드리스 / CI 대응:** 오디오 출력이 없는 더미 드라이버 상태에서도 `AudioServer`가 항상 존재하므로, 기기 출력이 없는 헤드리스 모드에서 오디오 풀을 생성하고 `play()`를 호출하더라도 에러가 발생하지 않습니다. 이는 `tests/run_audio_tests.gd` 및 UI 스모크 테스트를 통해 검증되었습니다.
- **설정 유지:** 볼륨 값(선형 0~1) 및 음소거 여부는 로컬 기기의 `user://settings.cfg` 파일의 `[audio]` 섹션에 저장되며 게임 실행 시 복원됩니다.

## 볼륨 / 음소거 조작 UI

`scripts/ui/SoundControl.gd` 모듈은 `Audio`와 직접 연결된 음소거 토글 버튼 및 볼륨 조절 슬라이더 UI를 구성합니다. 이 컴포넌트는 **로비**(매치 시작 전 진입 가능)와 게임 진행 중의 **일시정지 화면**에 삽입됩니다. 프로젝트의 UI 약속([`BRAND.md`](BRAND.md) §4 참고)을 따릅니다:

- 번역 카탈로그(`localization/ui_strings.csv`)를 활용하여 모든 레이블(`Sound`, `Mute`, `Volume`)을 지원하는 두 로케일(`en`, `ko`)로 제공하며, 코드에 문자열을 하드코딩하지 않습니다.
- UI 상태: 활성화 / 비활성화(음소거 시 슬라이더가 비활성화되고 포커스를 잃음) / 포커스 상태를 지원하며, 고대비 테마에 맞춘 포커스 링이 표시됩니다. UI 생성 시 현재의 볼륨 및 음소거 설정값이 반영됩니다.
- 접근성: 각 조작부에는 접근성 이름 및 설명(`A11y.label`)이 부여되어 있으며, 고대비 테마 하에서 키보드로 조작 가능합니다.
- 의미에 따른 색상 구분: 새로운 브랜드 컬러를 도입하지 않고, 기존의 기능적인 테마(`assets/theme.tres`)를 상속하여 사용합니다.

## 출처 및 라이선스

**게임에 포함된 모든 효과음은 100% 독창적인 소스입니다.** 어떠한 형태의 샘플러, 외부 레코딩 또는 서드파티 에셋도 사용하지 않았습니다. 9개의 모든 WAV 파일은 NumPy 오실레이터와 노이즈 엔벨로프를 활용하여 `assets/audio/generate_sfx.py` 스크립트를 통해 절차적으로 합성되었습니다. 이 생성 스크립트는 저장소 내에 보존되어 에셋 생성 파이프라인의 재현성과 검증 가능성을 제공합니다. 소리가 기본적인 합성 원리로부터 직접 생성되었기 때문에, 제3자의 저작권 소송 우려가 전혀 없으며 **상업적 이용에도 안전**합니다 — 이는 브랜드의 독창성 지향점과 일치합니다.

효과음 생성 스크립트를 수정한 후 다시 에셋을 생성하려면 아래 명령을 실행합니다:

```sh
python3 assets/audio/generate_sfx.py
```

"$GODOT" --headless --path . --editor --quit                       # 임포트/클래스 캐시 빌드
