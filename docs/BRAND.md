# 숨고 아케이드 — 브랜드 및 독창성 가이드

승인된 기회 요약서(Opportunity Brief)에서 요구하는 게임의 **브랜드 방향성** 및 **독창성/혼동 방지 체크리스트**에 대한 단일 진실 공급원(SSOT)입니다. 장르의 *규칙*은 저작권 보호 대상이 아니지만, *특정* 게임의 이름, 로고, 캐릭터, 아트, 음악은 보호 대상이므로 플레이어에게 노출되는 모든 요소는 100% 독창적으로 정의되며 기존 프랜차이즈와의 유사성을 피하도록 설계되었습니다.

관련 문서: 캐릭터 정체성 및 능력치 상세 정보는 [`docs/characters.md`](characters.md)에, 스프라이트 아트 제작 공식(그레이스케일 + 런타임 팀 컬러 마스크, 실루엣 메모 기반)은 [`docs/character-art.md`](character-art.md)에, 근거리 멀티플레이어 정보는 [`docs/NETWORKING.md`](NETWORKING.md)에 있습니다. 본 문서는 이들을 아우르는 **브랜드 수준의 상위 가이드**입니다.

## 1. 컨셉 및 세계관 설정

**숨고 아케이드.** 동네 **서비스 고수("고수")** — 청소, 이사, 인테리어 전문가, 과외 교사, 반려동물 케어 전문가 — 들이 업무 시간 이후에 스트레스를 푸는 가상의 물풍선 아레나입니다. 숨고 플랫폼 세계(지역 고수 매칭 마켓플레이스)는 오직 *분위기와 허구적 설정*으로만 사용됩니다. 실제 숨고 서비스의 묘사가 **아니며**, 게임은 공식 제품, 보증 또는 파트너십을 절대 암시해서는 안 됩니다.

## 2. 독창적인 캐릭터 라인업 (SSOT: `scripts/core/Characters.gd`)

실제 숨고 **서비스 카테고리**를 테마로 한 5명의 독창적인 "고수" 캐릭터가 등장합니다 (특정 인물이나 기존 아케이드 캐릭터가 아님). 각 캐릭터는 기존 `PlayerState` + `PowerUp` 모델을 기반으로 하며 `Spec` 한계값에 의해 제한되는 **고유한 시작 능력치 프로필**(사거리 / 물풍선 개수 / 이동 속도)을 가집니다 — **시작 프로필 외의 추가적인 밸런스 수치는 없으며, 매치 중 파워업을 통해 모두 동일한 한계치로 수렴하게 됩니다.** 전체 테이블 및 유사성 비교표는 [`docs/characters.md`](characters.md)를 참고하세요.

| 캐릭터 | 서비스 모티프 | 아키타입 (시작 프로필) |
| --- | --- | --- |
| **Sudsy (쓱싹이)** | 청소 | 스피드스터 (Speedster) |
| **Tote (짐꾼이)** | 이사 | 봄버 (Bomber, 추가 물풍선) |
| **Rolly (롤러)** | 인테리어 | 블래스터 (Blaster, 추가 사거리) |
| **Menty (멘티)** | 레슨 | 올라운더 (All-rounder) |
| **Paws (발바닥)** | 반려동물 케어 | 스커미셔 (Skirmisher) |

모든 이름/모티프/개성 대사는 `tr()`을 통해 해석되는 다국어 번역(i18n) **키**로 관리되며, 절대 코드에 하드코딩되지 않습니다.

## 3. 독창성 및 혼동 방지 체크리스트

| # | 확인 사항 | 상태 |
| --- | --- | --- |
| 1 | **이름**이 독창적이며 기존 프랜차이즈 타이틀과 중복되지 않음 | ✅ "Soomgo Arcade" (숨고 아케이드) |
| 1 | **로고/아이콘**이 특정 프랜차이즈의 캐릭터나 로고가 아닌 독창적이고 범용적인 마크임 | ✅ `icon.svg` (추상적인 형태) |
| 1 | **캐릭터** (이름, 모티프, 외형)가 기존 라인업과 유사하지 않음 | ✅ Sudsy/Tote/Rolly/Menty/Paws — 서비스 직종 기반이며, 기존 아케이드 IP의 토끼나 동물 캐릭터들과 다름 (characters.md §2 참고) |
| 1 | **기술/능력치 표현 방식**이 기존 작품들을 연상시키지 않음 | ✅ 아키타입 이름 명명(스피드스터/봄버 등) 및 독창적인 모티프 소품 사용 |
| 1 | **맵 테마**가 기존 프랜차이즈 리소스를 쓰지 않고 범용 격자/벽/블록 모양을 사용함 | ✅ `Game.gd` 내 단색 타일 적용 |
| 1 | **음악** — 도용되거나 파생된 오디오를 탑재하지 않음 | ✅ 배경음악 없음. 포함된 **효과음**은 100% 독창적임 (절차적 합성 방식, [`docs/audio.md`](audio.md) 참고) |
| 2 | 보호받지 못하는 **규칙/메커니즘**(물풍선, 사거리, 속도, 연쇄 폭발, 갇힘/구출)만 재사용 | ✅ `Spec.gd` / `Simulation.gd` |
| 2 | 메커니즘은 장르 공통적일 수 있으나 **이름/모티프/효과는 독창적**임 | ✅ 능력치 프로필은 `PowerUp` 모델을 재사용하되 명칭은 독창적으로 정의 |
| 4 | 숨고 세계관은 **허구**이며, "공식/제휴" 관계를 암시하지 않음 | ✅ 실제 서비스가 아닌 서비스 고수 아케이드 캐릭터로 한정하여 표현 |
| 5 | 각 캐릭터는 기존 모델에서 **1개 이상의 구별되는 특성**을 가짐 | ✅ 5가지 고유한 시작 능력치 프로필 보유 |

> **"창작 방지" 가드레일:** 캐릭터 수(목표치인 4~6개 범위 내인 5개), 캐릭터별 능력치 프로필, 그리고 이름만이 임의로 정의된 값이며, 각 능력치는 기존 `Spec` 한계값 내에 존재합니다. 팔레트 토큰이나 로케일 세트는 새로 만들어내지 않았습니다. 향후 저장소에 디자인 SSOT(팔레트 토큰, 지원 로케일 세트, 밸런스 사양)가 추가된다면, 여기서 임의로 값을 생성하는 대신 해당 명세를 이 승인 기준에 통합해야 합니다.

## 4. UI / 접근성 약속 (브랜드 접점 영역)

본 저장소는 신규 프로젝트로 **디자인 SSOT가 존재하지 않습니다** (검색 대상: `*Tokens*`, `theme.*`, `tokens.json`, `tailwind.config.*`, CSS 변수, `DESIGN*.md`, `CLAUDE.md`/`AGENTS.md`, `*.po`/`*.strings`/`messages/*`). 기획서에 따라 브랜드 관련 UI에는 **보편적인 UX 및 접근성만 적용**하며, 자체적인 브랜드 팔레트나 고정된 로케일 세트를 임의로 생성하지 않습니다:

- **모든 문자열 외부화**: 모든 캐릭터 이름, 모티프, 개성 대사와 게임 내 `HUD_CHARACTER` 레이블을 포함하여 모든 문자열을 `localization/ui_strings.csv` → `tr("KEY")`로 처리합니다. 기획서에서 승인한 로케일 세트인 영어(`en`)와 한국어(`ko`) 열이 제공되며, 코드 수정 없이 CSV 열을 추가하는 것만으로 새로운 로케일을 추가할 수 있습니다. 시스템 설정이 한국어일 때 한국어가 자동 선택되며 (명시적인 언어 전환 기능인 `Strings.toggle_language` 포함), 번역이 누락되거나 비어 있는 경우 영어가 기본값(Fallback)으로 적용됩니다.
- **캐릭터 화면 요소**: 로비의 슬롯별 캐릭터 `OptionButton`과 **게임 내 HUD 캐릭터 정보 표시 영역**(`Game.gd`)은 포커스가 가능하며, 접근성 레이블/설명(`A11y.gd`)이 제공되고, 고대비 테마 및 시각적 포커스 링(`assets/theme.tres`)을 상속받습니다.
- **의미에 따른 색상 구분**: 아레나의 색상은 기능적(플레이어/팀 구분)으로만 사용되며 브랜드 팔레트가 아닙니다 — 이는 `Game.gd` 및 `characters.md`에 문서화되어 있습니다. 또한 색상이 **유일한 단서가 되지 않도록** 합니다. 물풍선에 갇힌 상태, 방금 구출된 상태/무적 상태를 포함하여 보드의 모든 마커에는 팀별 고유한 **도형 마커**(`scripts/ui/TeamMarker.gd`)를 겹쳐서 표시하며, HUD 점수판 및 조작 바에서도 이를 재사용합니다. 이를 통해 색약 플레이어도 최대 8개 팀 중에서 아군과 적군을 색상에만 의존하지 않고 쉽게 구별할 수 있습니다.

## 5. 검증

```sh
GODOT=/Applications/Godot.app/Contents/MacOS/Godot
"$GODOT" --headless --path . --editor --quit                     # 임포트/클래스 캐시 빌드
"$GODOT" --headless --path . --script res://tests/run_tests.gd   # 규칙 + 라인업 + 프로필 검증
"$GODOT" --headless --path . --script res://tests/smoke_ui.gd    # 로비/게임/연결 UI 검증
"$GODOT" --headless --path . --script res://tests/run_audio_tests.gd  # 렌더링 측 SFX 레이어 검증
```
